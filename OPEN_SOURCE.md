# System Design Assistant — OPEN SOURCE

Open-core (D-007); lighter OSS footprint than the dev-tool projects.

**Open (Apache/MIT):** ADR/diagram templates, Mermaid generation helpers, a system-design pattern library, and prompt/rubric guidelines for design + interview scoring. These build trust + SEO + community without giving away the product.

**Proprietary:** the hosted assistant, codebase-grounding integration (via #2), design memory, multi-agent review, interview-prep engine + scenario library + scoring, teams/governance, ContextOS integration.

**Rationale:** templates + pattern libraries are good open/SEO assets (devs find + star them); the *grounded, persistent, interactive* product — especially grounding (#2) and the prep engine — is the value. Interview scenarios/rubrics are proprietary content (the moat for the prep market).

**Licensing:** OSS Apache/MIT; product commercial; CLA. **Fork risk:** low — the moat is grounding + memory + prep content + reasoning quality, not templates. Community contributes patterns/templates (good funnel).
