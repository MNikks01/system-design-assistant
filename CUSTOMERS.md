# System Design Assistant — CUSTOMERS

## ICPs
- **ICP-1: Senior engineers / architects** designing + documenting systems. Want grounded design + ADRs + diagrams. Pro/Team.
- **ICP-2: Engineering teams** wanting shared, persistent design memory + review. Team (ContextOS module).
- **ICP-3 (distinct market): Interview-preppers** practicing system-design interviews. Prosumer Pro; high-intent, time-boxed.
- **ICP-4: Enterprise architecture teams** — standards/governance/ADR-at-scale. Enterprise (likely via ContextOS).
- **ICP-5 (free→Pro): individual devs** exploring designs / learning.

## Anti-personas
Non-technical; teams with no design/documentation discipline interest; tiny systems.

## Buyer vs. user
User = engineer/prepper. Buyer = the engineer (Pro/prep), eng lead (Team), architecture lead (Enterprise via #1). Two motions: prosumer self-serve (interview prep) + B2B (via ContextOS).

## Prioritization
Interview-prep (fast prosumer revenue + funnel) + individual designers (launch) → teams/design memory (Y1, ties to #2) → enterprise architecture (Y2, via #1).
